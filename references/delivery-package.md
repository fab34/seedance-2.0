# Delivery Package

The standard shape of a final deliverable. A user who receives only a prompt string still has to trust it blind; a user who receives the package can see the film before generating it, react to the plan instead of the output, and reuse the plan when a retake is needed. The package wraps the prompt - it never inflates it: the final Seedance prompt stays compact, natural language, and inside the verified active-surface budget regardless of how full the package around it is.

## Full Package

Deliver all six parts for production requests, professional users, sequence projects, or whenever the user asked for planning, structure, or suggestions:

1. **Final prompt.** Generation-ready, pasteable as-is, in the target language, under the active-surface budget, anti-slop passed. This is the headline of the message, not an appendix. Include the optional compressed or translated variant when the surface or user benefits.
2. **Shot and camera plan.** One row per shot: shot size, angle, the single primary camera move with start and endpoint, and what the move is motivated by. For a single-shot clip this is one row, not a table ceremony.
3. **Framing and composition notes.** Where the subject sits in frame, headroom or lead room, foreground/background layers, and the aspect-ratio consequence for the target platform.
4. **Scene and light.** The world of the shot: location, time of day, one useful environmental detail, the motivated light source, and palette or style anchor.
5. **Script / beat plan.** Before state, visible action, changed final state per the storytelling framework - plus dialogue lines and sound intent (ambience, SFX, music, or silence) with rough timing. For sequences, the per-clip job list with each clip's completed endpoint.
6. **Story structure suggestion.** One short paragraph: why the beats sit in this order, where the turn lands, and - proactively - what the natural growth path is (a stronger ending, a part two, a three-clip arc) so the user hears the option before asking. Raise sequence expansion here, after the first draft, never as an intake question.

Close the package with the assumption switches (`I assumed dusk - say "noon" and I'll relight it`) and the next step (generate now, swap an assumption, or expand to a sequence).

## Compact Package

For fast-lane single clips and casual users, compress without dropping the spine: final prompt, then one line each for camera, framing, scene, and beat, then one assumption-switch line and one alternate-direction sentence. The compact package fits in roughly ten lines. Never deliver a bare prompt with no plan at all - even the compact package lets the user see the shot before spending a generation.

## Consistency Rule

Every part of the package describes the same film. The shot plan may not contain a move the prompt does not request; the beat plan may not promise an ending the prompt does not reach; the framing note may not contradict the aspect ratio. Before delivery, read the package once as a whole and fix any part that drifted. The package is derived from the internal brief - it is the brief made visible, not a second creative pass.

## Register

Parts 2-6 are written in the user's register: plain words for a non-expert (`camera starts wide and slowly moves in until the label fills the screen`), director language for a professional (`slow dolly-in, MS to MCU, motivated by the reveal`). The final prompt itself always uses full production language - the model is the one reader who never needs the plain version.
